## Problem Statement

NNAI는 AI 기반 디지털 노마드 이민 설계 서비스로, 백엔드/프론트엔드 모두 배포 완료 상태. 하지만 수익 모델이 Google AdSense 하나뿐이고, 실질적 매출은 0에 가깝다. 해커톤 프로젝트에서 실제 수익을 만드는 비즈니스로 전환해야 한다.

## What Makes This Cool

"추천 -> 실행"까지의 전체 파이프라인을 하나의 서비스가 커버한다는 점. 도시 추천, 비자 안내, 세금 경고, 숙소 연결, 보험 추천까지 원스톱. 대부분의 노마드 도구는 "정보 제공"에서 끝나지만 NNAI는 "행동 전환 지점"에 수익을 삽입할 수 있는 구조를 이미 갖고 있다.

## Constraints

- 해커톤 프로젝트 — 빠른 실행이 핵심
- 2인 팀 (개발자 + 기획)
- 백엔드 API 스키마 변경은 별도 협의 필요 (CLAUDE.md 규칙)
- 프론트엔드 완성도가 아직 낮음 (Step 1 입력 폼, 결과 렌더링 미완)
- MAU 0 상태 — 트래픽 확보가 수익화보다 선행되어야 할 수 있음

## Premises

1. **한국 디지털 노마드 시장은 수익화할 만큼 성장 중이다.** 경쟁이 거의 없고, 한국어 우선 서비스는 NNAI가 유일. 니치지만 전환율이 높은 시장.

2. **어필리에이트가 첫 수익 채널로 적합하다.** accommodation.py에 딥링크가 이미 있고, SafetyWing/Airalo 같은 노마드 특화 제품은 타겟이 정확히 일치.

3. **Step 2 상세 가이드에 프리미엄 게이팅이 가능하다.** Step 1 무료 추천 -> Step 2 상세 가이드의 가치 차이가 명확하면 결제로 전환됨.

4. **기존 코드베이스가 수익화 인프라를 빠르게 지원한다.** OAuth, Pin, 페르소나, 쉥겐 계산기, 세금 경고 등 이미 구축된 기능 위에 Stripe + 게이팅 로직만 추가.

## Cross-Model Perspective

SECOND OPINION (Claude subagent):

**가장 멋진 버전: "Nomad Autopilot"**
일회성 추천이 아닌, 실시간 적응형 재배치 엔진. 비자 카운트다운, 환율 변동, 세금 거주지 임계값을 기반으로 매주 리밸런싱하는 살아있는 여정. 이미 쉥겐 계산기 + 세금 경고 + 도시 점수가 있으니, 이걸 사용자별 영속 타임라인으로 연결. 타로 카드 UI가 조건 변화에 따라 재셔플되는 살아있는 카드 패가 된다.

**가장 흥미로운 포인트:** 파트너 광고 모듈 프로토타입과 accommodation.py 딥링크. 콘텐츠 사이트가 아니라 노마드 서비스의 "유통 채널"을 만들고 있다.

**50% 달성 도구:** Cal.com (오픈소스 예약 플랫폼). 나머지 50%는 매칭 엔진 + 제공자 온보딩.

**주말에 만들 것:** Step 2 프리미엄 게이트 + Stripe. ~300줄. 일요일 밤 배포, 50명에게 전달, 금요일에 결제 의향 검증 완료.

**Cross-model synthesis:**
- 동의: Step 2 프리미엄 게이팅이 가장 빠른 PMF 검증 수단
- 동의: 이 프로젝트의 본질이 "유통 채널"
- 추가: "Nomad Autopilot"은 Phase 4+ 로드맵에 포함
- Cal.com 연동은 마켓플레이스 단계(BM-3)에서 활용 가능

## Approaches Considered

### Approach A: Affiliate-First (최소 실행)

Summary: 기존 딥링크에 affiliate ID 추가 + Step 2에 "실행 가이드" 섹션 삽입
Effort: S (1-2일)
Risk: Low
Completeness: 6/10

Pros:
- 코드 변경 최소 (accommodation.py, parser.py, AdModule.tsx 3개 파일)
- 파트너 승인 대기 중에도 링크 구조 완성 가능
- 트래픽 0이어도 구조가 준비되어 있으면 즉시 수익 시작 가능

Cons:
- MAU 낮으면 월 수십 달러 수준
- 결제 의향(willingness-to-pay) 검증 불가
- 수동적 수익 - 성장 동력이 약함

Reuses: accommodation.py, parser.py format_step2_markdown(), AdModule.tsx

### Approach B: Freemium Gate + Affiliate 동시 (이상적 아키텍처)

Summary: Stripe 연동 + Step 2 게이팅(Free: 월 2회) + affiliate 동시 구축
Effort: M (5-7일, reviewer 피드백 반영)
Risk: Med
Completeness: 8/10
**Pro 가격: $9.99/월 (약 13,000원)**

Pros:
- 결제 의향 직접 검증 (PMF의 가장 강력한 신호)
- 두 채널 동시 수익화
- 유료 전환 데이터 기반으로 제품 방향 결정 가능

Cons:
- Stripe 연동(웹훅+에러처리+멱등성) 실제 200-400 LOC
- 프론트엔드 결과 페이지 기본 플로우가 선행 조건
- **한국 개인 계정 Stripe recurring billing 제한** - 사업자등록 또는 Paddle/Lemon Squeezy 대안 필요

Reuses: Google OAuth (사용자 식별), utils/db.py (users 테이블), 기존 /api/detail 엔드포인트

### Approach C: 페르소나 기반 큐레이션 커머스 (창의적)

Summary: 5가지 페르소나별 "노마드 스타터 킷" 번들 - 보험+eSIM+코워킹 패스
Effort: M (4-7일)
Risk: Med-High
Completeness: 7/10

Pros:
- 기존 페르소나 시스템(5종)을 직접 수익화
- 높은 객단가 ($50-100/번들 vs $10/월)
- "타로 카드로 페르소나 진단 -> 맞춤 킷" 스토리텔링

Cons:
- 파트너별 개별 협의 필요
- 번들 구성/가격 검증이 복잡
- 재구매율 불확실

Reuses: utils/persona.py (5종 페르소나), 타로 카드 UI, 커뮤니티 핀

## Recommended Approach

## Open Questions & Blockers

1. **[HARD BLOCKER] Stripe 한국 제한:** 한국 개인 계정은 recurring billing 제한. 사업자등록 필요 여부 확인 필수. **대안: Paddle, Lemon Squeezy** (한국 개인 가능, MoR 모델). Phase 2 착수 전 결제 플랫폼 확정 필요.
2. **프론트엔드 선행 조건:** Step 1 결과 렌더링 + Step 2 가이드 UI 기본 플로우 동작이 Week 2 착수 조건. 현재 미완성.
3. **MAU 확보 전략:** SEO (한국어 "디지털 노마드 비자" 키워드), 노마드 커뮤니티 (디노공, 해외살이 카페), 인스타/유튜브 콘텐츠. Phase 1은 MAU 0에서도 인프라 준비 가치 있음.
4. **Affiliate 승인 소요:** SafetyWing, Airalo 등 보통 1-3일.
5. **HTTP 상태 코드:** 402 (Payment Required)는 실험적. 403 + `X-Payment-Required: true` 헤더 대안 고려.
6. **Webhook 실패 시 fallback:** Stripe 웹훅 실패 시 사용자가 결제했는데 Pro 미활성화되는 케이스. polling 또는 checkout session 확인 엔드포인트 필요.

## What I noticed about how you think

- "실제 수익"을 강조한 것이 인상적. 이론적 BM이 아니라 돈이 들어오는 구조를 원한다는 명확한 의도.

- "묻지 마세요", "멈추지 말고 쭉 진행해줘"... 실행 속도를 극도로 중시. 분석 마비에 빠지지 않는 빌더.
